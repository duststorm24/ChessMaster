Math Notes
IK Summary
Base yaw is solved from atan2(y, x), with +X pointing East, +Y pointing North, and home facing East. The arm plane itself is rendered 180 deg behind that heading so the links sit on the west side when the base faces east.
XYZ is reduced to planar arm math using radial reach r = sqrt(x^2 + y^2) and vertical offset z - 16.03.
Arm 2 true home is modeled folded back upward toward the origin, so its effective home-relative IK angle is 180 - 14.76 = 165.24 deg.
The blue reachable volume is sampled from real joint travel assumptions: Base 0 to 270 deg, Arm 1 from 4.76 to 191.00 deg, Arm 2 relative from -22.76 to 165.24 deg.
The IK solver checks both behind-the-base branches and now prefers the candidate closest to the previous pose, which prevents the random 180 deg base flip when you move the target continuously.
This page never sends robot motion commands. It stays focused on learning and visualization, although you can save a drawn line into the separate Control Panel line library for later testing.
